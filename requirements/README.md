## For Claude

When working on this project:

1. **Read BUSINESS-REQUIREMENTS.md first** - Platform overview and features
2. **Use DOCTYPES.md** - For exact field specifications when creating DocTypes
3. **Use BUSINESS-LOGIC.md** - For calculation logic and API implementations
4. **Follow the workflows** - Booking flow, lister onboarding
5. **Implement notifications** - Email triggers defined in BUSINESS-LOGIC.md

## Adding New Requirements

When Shilpa provides new requirements:
1. Update BUSINESS-REQUIREMENTS.md for feature overview
2. Update DOCTYPES.md if schema changes
3. Update BUSINESS-LOGIC.md if new calculations/APIs needed
4. Update this README if adding new docs
